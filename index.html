Часто задаваемые вопросы по тестированию.

1. Какие бывают типы HTTP запросов?

Основные типы HTTP запросов:

  • –
    Запрос GET. На получение данных. Не используется в Body, является безопасным, идемпотентным и кешируемым.
  • –
    Запрос POST. На отправку данных. Используется в Body, является небезопасным, не идемпотентным и не кешируемым.
  • –
    Запрос PUT. На изменение данных. Используется в Body, является небезопасным, идемпотентным и не кешируемым.
  • –
    Запрос DELETE. На удаление данных. Используется в Body, является небезопасным, идемпотентным и не кешируемым.
  • –
    Запрос PATCH. На частичное изменение данных. Используется в Body, является небезопасным, не идемпотентным и не кешируемым.
2. Какие статусы HTTP ответа бывают?

Вообще статусов очень много, но вот основные:

  • –
    100-199 — информационные статусы.
  • –
    200-299 — успешные (сервер получил и принял запрос).
    200 — успех (OK)
    201 — создано (Created)
  • –
    300-399 - перенаправление (редирект).
    301 — адрес переехал на другой домен (Moved Permanently)
  • –
    400-499 клиентские ошибки.
    400 — неверный запрос (Bad Request)
    401 — не авторизован (Unauthorized)
    403 — нет доступа (Forbidden)
    404 — не найдено (Not Found)
  • –
    500-599 серверные ошибки.
    500 — внутренняя ошибка сервера (Internal Server Error)
    504 — таймаут, то есть вышло время ожидания ответа (Gateway Timeout)
3. В чем разница между REST и SOAP?
  • –
    REST – это архитектурный стиль, правила которого желательно соблюдать
    1. REST поддерживает различные форматы (JSON, XML, текстовые форматы)
    2. REST работает только по протоколу HTTP (или HTTPS)
    3. Согласно REST часть запросов может быть закэширована
    4. Документация обычно в Swagger или в Open API формате
    5. Инструмент для тестирования — Postman
  • –
    SOAP – это протокол, используя который важно строго соблюдать все его правила
    1. SOAP поддерживает только формат XML
    2. SOAP может работать с различными протоколами (HTTP, SMTP, FTP)
    3. В SOAP не предусмотрено кэширование
    4. Документация в WSDL (вэ-эс-дэ-элька)
    5. Инструмент для тестирования — SOAP UI
4. Какие типы данных есть в JSON?
  • –
    Строка.
  • –
    Число.
  • –
    Boolean.
  • –
    null.
  • –
    Массив [].
  • –
    Объект {}.
5. Что такое API и в каких сервисах или форматах может быть описана документация по API?

API — это интерфейс, с помощью которого можно взаимодействовать с бэкендом. Документацию описывают обычно в:

  • –
    Swagger.
  • –
    Вики.
  • –
    Redoc.
  • –
    WSDL для SOAP протокола.
